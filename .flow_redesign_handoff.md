- `sub_scenario_chd` appears to be attached to `m_sub_wells` while later cells use

  may be intended.

## Transport Workflow Findings

The transport notebook is not a simple local-refinement pattern that can be copied
into flow. It is MF6/DISV and helper/cache driven:

- calibrated MF6/DISV base is loaded via `model_io_utils.ensure_flow_model()`
- local refinement uses `_SUPPORT/src/model_io_utils.py::build_refined_gwf_model()`
- transport scenario uses `_SUPPORT/src/transport_base_model.py::build_spill_scenario()`
- outputs are cached in `base_cache.npz`
- notebook loads cache unless `FORCE_RERUN=True`
- refinement is described to students as instructor-provided/validated and fragile
- helper includes retry logic for refinement failures/SIGILL/Triangle-type issues

Key conclusion: transport local refinement is simpler for students only because the
fragile parts are hidden, validated, and cached. Student-built local refinement is
not clearly simpler.
